# üîß Device Setup Guide - iOS & Android\n\n**Complete setup instructions for testing on real devices**\n**Date:** December 9, 2025\n\n---\n\n## üçé iOS Device Setup\n\n### Prerequisites\n- Mac computer (required for iOS)\n- iPhone (iOS 14+)\n- USB cable (Lightning or USB-C)\n- Xcode (can be installed via App Store or Command Line Tools)\n\n### Step-by-Step Setup\n\n#### 1. Install Xcode (if not already installed)\n\n**Option A: Via App Store (Recommended)**\n```bash\n# Open App Store and search for \"Xcode\"\n# Download and install (about 12 GB)\n# This takes 15-30 minutes\n```\n\n**Option B: Via Command Line (Faster)**\n```bash\n# Install Command Line Tools only\nxcode-select --install\n\n# This is smaller (500 MB) and faster\n# Click \"Install\" when prompted\n# Takes 5-10 minutes\n```\n\n**Verify Installation:**\n```bash\nxcode-select -p\n# Should output: /Applications/Xcode.app/Contents/Developer\n# Or: /Library/Developer/CommandLineTools\n```\n\n#### 2. Connect iPhone to Mac\n\n1. **Plug in USB cable**\n   - iPhone to Mac\n   - Use original Apple cable for reliability\n\n2. **Unlock iPhone**\n   - Face ID, Touch ID, or passcode\n   - Keep it unlocked during setup\n\n3. **Trust the Computer**\n   - You'll see: \"Trust this computer?\"\n   - Tap \"Trust\"\n   - Tap \"Allow\" (if prompted for sensitive info)\n\n4. **Verify Connection**\n   ```bash\n   # In terminal, check if Mac sees iPhone\n   system_profiler SPUSBDataType | grep -i \"iPhone\\|iPad\"\n   \n   # Or use Xcode:\n   # Xcode ‚Üí Window ‚Üí Devices and Simulators\n   # You should see your iPhone listed\n   ```\n\n#### 3. Check iOS Version\n\n1. **On iPhone:**\n   - Settings ‚Üí General ‚Üí About\n   - Note iOS version (should be 14+)\n   - Target: iOS 15 or higher for best compatibility\n\n2. **Verify with Xcode:**\n   ```bash\n   # Xcode ‚Üí Window ‚Üí Devices and Simulators\n   # Select your device\n   # Check \"OS Version\" field\n   ```\n\n#### 4. Enable Developer Mode (iOS 16+)\n\n1. **On iPhone:**\n   - Settings ‚Üí Privacy & Security\n   - Scroll down\n   - Toggle \"Developer Mode\" ON\n   - Tap \"Restart\"\n   - iPhone will restart\n\n2. **Trust the Computer Again**\n   - When phone restarts\n   - Tap \"Trust\" when asked\n\n#### 5. Test Connection\n\n```bash\n# Terminal command\nsudo xcode-select --install\n\n# If you get: \"xcode-select: error: command line tools are already installed\"\n# Good! You're ready.\n\n# Check devices\nxcode-select -p\n```\n\n#### 6. Ready to Build!\n\n```bash\ncd /Users/chinmaybehera/memovox-rel1/memovox-mobile\nnpm run ios\n```\n\n**What happens:**\n1. Builds app (5-10 minutes first time)\n2. Installs to iPhone\n3. Opens automatically\n4. Ready to test!\n\n---\n\n## ü§ñ Android Device Setup\n\n### Prerequisites\n- Mac or Linux computer\n- Android phone (Android 8+)\n- USB cable (USB-C or Micro USB)\n- Java SDK (usually auto-installed by Android SDK)\n\n### Step-by-Step Setup\n\n#### 1. Enable Developer Mode on Phone\n\n1. **Open Settings**\n   - Navigate to \"Settings\"\n\n2. **Find About Phone**\n   - Scroll to \"About Phone\"\n   - OR: Look for \"Phone Info\"\n\n3. **Find Build Number**\n   - Look for \"Build Number\" or \"Build Version\"\n   - Tap it 7 times (rapidly)\n   - You'll see: \"Developer Mode enabled\"\n   - Or: \"You are now a developer!\"\n\n4. **Go Back to Settings**\n   - You should now see \"Developer Options\"\n   - (Usually near the bottom)\n\n#### 2. Enable USB Debugging\n\n1. **Open Developer Options**\n   - Settings ‚Üí Developer Options\n   - (If not visible, see step 1.3)\n\n2. **Toggle USB Debugging**\n   - Find \"USB Debugging\"\n   - Toggle it ON\n   - You'll see: \"Allow USB debugging?\"\n   - Tap \"OK\"\n\n3. **Toggle USB File Transfer**\n   - Find \"USB File Transfer\" or \"File Transfer Mode\"\n   - Toggle it ON\n\n#### 3. Check Android Version\n\n1. **On Phone:**\n   - Settings ‚Üí About Phone\n   - Look for \"Android Version\"\n   - Target: Android 8+ (8, 9, 10, 11, 12, 13, 14)\n\n2. **Verify with ADB:**\n   ```bash\n   adb shell getprop ro.build.version.release\n   # Should output: 8, 9, 10, 11, 12, 13, or 14\n   ```\n\n#### 4. Install Android SDK Tools\n\n**Check if already installed:**\n```bash\nls ~/Library/Android/sdk/platform-tools/adb\n\n# If exists, you're good!\n# If not, install:\n```\n\n**Install Android SDK:**\n\n**Option A: Via Homebrew (Easiest)**\n```bash\n# Install Android SDK Manager\nbrew install android-commandlinetools\n\n# Or using Expo CLI (auto-manages Android)\nexpo install-android-sdk\n```\n\n**Option B: Manual Installation**\n1. Download from: https://developer.android.com/studio\n2. Extract to: `~/Library/Android/sdk`\n3. Add to PATH (see step 4)\n\n**Add to PATH:**\n```bash\n# Open ~/.zshrc (or ~/.bash_profile if using bash)\nvim ~/.zshrc\n\n# Add these lines:\nexport ANDROID_HOME=$HOME/Library/Android/sdk\nexport PATH=$PATH:$ANDROID_HOME/emulator\nexport PATH=$PATH:$ANDROID_HOME/platform-tools\n\n# Save and exit (ESC, :wq)\n# Reload:\nsource ~/.zshrc\n\n# Verify:\nadb --version\n# Should show version number\n```\n\n#### 5. Connect Android Phone\n\n1. **Plug in USB Cable**\n   - Phone to Mac\n   - Use quality cable\n   - Avoid cheap cables (connection issues)\n\n2. **Choose File Transfer Mode**\n   - Phone shows: \"USB connected\"\n   - Tap the notification\n   - Select: \"File Transfer\" or \"Transfer Files\"\n   - (Not \"Charge Only\")\n\n3. **Trust the Computer**\n   - Phone may ask: \"Allow USB debugging?\"\n   - Tap \"Allow\"\n   - Check \"Always allow from this computer\" (optional)\n\n4. **Verify Connection**\n   ```bash\n   # List connected devices\n   adb devices\n   \n   # You should see:\n   # List of attached devices\n   # emulator-5554          device\n   # R38K20BCKKZ           device (your phone's ID)\n   # \n   # If you see \"offline\", disconnect and reconnect\n   # If you don't see your device, try USB Debugging again\n   ```\n\n#### 6. Test Connection\n\n```bash\n# Run a simple command\nadb shell getprop ro.model.name\n\n# Should output your phone model\n# Example: SM-G900F (Samsung Galaxy S5)\n```\n\n#### 7. Ready to Build!\n\n```bash\ncd /Users/chinmaybehera/memovox-rel1/memovox-mobile\nnpm run android\n```\n\n**What happens:**\n1. Builds app (5-10 minutes first time)\n2. Installs to phone\n3. Opens automatically\n4. Ready to test!\n\n---\n\n## üîó Connection Troubleshooting\n\n### iPhone Connection Issues\n\n#### \"Device Not Found\"\n```bash\n# 1. Disconnect USB\n# 2. Try different USB port (prefer back ports on Mac)\n# 3. Try different USB cable\n# 4. Restart iPhone\n# 5. Reconnect\n# 6. Tap \"Trust\" on iPhone\n```\n\n#### \"Trust Error\" Dialog\n```bash\n# On iPhone, tap \"Trust\"\n# Then immediately tap \"Allow\" if prompted\n# Then try building again\n```\n\n#### Xcode Says \"Device Locked\"\n```bash\n# On iPhone, unlock with Face ID or passcode\n# Keep it unlocked during build\n```\n\n#### Certificate/Provisioning Errors\n```bash\n# These are advanced iOS issues\n# For now, skip and use iOS Simulator instead:\nnpm run ios -- --simulator\n```\n\n### Android Connection Issues\n\n#### \"adb: command not found\"\n```bash\n# Add Android SDK to PATH\nexport PATH=$PATH:~/Library/Android/sdk/platform-tools\n\n# Permanent fix:\nvim ~/.zshrc\n# Add the line above\n# Save and source ~/.zshrc\n```\n\n#### \"offline\" or \"unauthorized\"\n```bash\n# 1. Disconnect USB\n# 2. Settings ‚Üí Developer Options ‚Üí USB Debugging (toggle OFF then ON)\n# 3. Reconnect USB\n# 4. Tap \"Allow USB debugging?\" on phone\n# 5. Try: adb devices\n```\n\n#### \"no permissions\"\n```bash\n# On Linux/Mac, run:\nsudo adb kill-server\nsudo adb devices\n\n# Or use Expo (handles permissions automatically):\nexp start\n# Press 'a' for Android\n```\n\n#### Device Appears but Build Fails\n```bash\n# Uninstall app first:\nadb uninstall com.memovox.app\n\n# Then rebuild:\nnpm run android\n```\n\n---\n\n## üñ•Ô∏è Alternative: Use
